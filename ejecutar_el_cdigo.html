Ejecutar el código

¿COMPILADOR O INTÉRPRETE?:

El procesador solo entiende el Lenguaje Maquina (lenguaje binario) por lo tanto necesita un elemento que le  permita traducir lo que el programador escriba en C/C++ a unos y ceros, C++ es un lenguaje compilado, por lo tanto necesita un compilador que haga este trabajo, en este proceso lo que sucede es que primero se revisa el código a través de su sintaxis y luego que se corrobore que todo está bien se lleva al lenguaje que el procesador entiende, en este proceso se generan dos archivos más además del .cpp se crea un .o que es el archivo de compilación y por ultimo un .exe que viene siendo el archivo ejecutable, escrito en unos y ceros.

Existen los lenguajes interpretados como PHP o Python que a diferencia de C/C++ no genera un programa ejecutado por el procesador sino por un intérprete por lo tanto el producto final no es necesariamente un archivo escrito en 1 y 0. En el caso de Python el archivo fuente es un archivo con extensión .py y el ejecutable es el mismo ya que solo necesita el intérprete de Python para correr, aunque el mismo puede ser empaquetado en otros tipos de archivos ejecutables, .exe para que sea ejecutado por el mismo procesador o .html para que pueda ser ejecutado por un navegador.
